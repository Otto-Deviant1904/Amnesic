# Threat Model

This document defines what Amnesic Browser is engineered to defend against,
what it explicitly does not attempt to defend against, and — for the things
it does defend against — exactly which mechanism is doing the work and what
that mechanism's limits are. Section 2 reflects corrections made after
verifying the original plan against the pinned dependency
(`electron@43.0.0`, Chromium 150.0.7871.46); see
`docs/adr/0002-electron-43-flag-and-api-corrections.md` for the reasoning
behind those corrections and `research/*.md` for the underlying source
citations.

## 1. Scope

| Threat                                                           | In scope?                                                 | Why                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ---------------------------------------------------------------- | --------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Forensic disk analysis after the app closes                      | **Yes — primary goal**                                    | This is the core promise: nothing recoverable from disk once the process exits.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| A live attacker with RAM access while the app is running         | Partial                                                   | RAM will always contain plaintext page content while browsing — no browser avoids this. We minimize residency time, we do not eliminate it.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| Network-level observer (ISP, Wi-Fi owner)                        | **Mitigated when proxy mode is enabled (Tor by default)** | Off by default, never persisted (see ADR 0007, generalized by ADR 0012). When enabled, traffic routes through a user-supplied proxy — SOCKS5 (the Tor default, `127.0.0.1:9050`), or an HTTP/HTTPS proxy a VPN or provider exposes. For **all three schemes** the destination hostname is resolved **at the proxy, never locally** (SOCKS5's domain-name address type; an HTTP proxy's absolute-form request target / `CONNECT`), and navigation **fails closed** if the proxy is unreachable (a bare `<scheme>://host:port` rule has no direct fallback). SOCKS4 is deliberately excluded — it would leak every hostname to the local resolver (ADR 0007 decision 3, ADR 0012). Per-scheme trust, stated plainly: **Tor / SOCKS5** uses a relay model, so no single operator sees both your real IP and your destinations; an **HTTP/HTTPS proxy** is a single operator (your VPN endpoint) who sees your real IP and can correlate all your traffic — transport privacy, **not** anonymity and **not** anti-fingerprinting, and never as private as Tor. General limits carry over: this is footprint elimination plus transport privacy, not anonymity parity with Tor Browser (no uniform fingerprint, no control-port/circuit-health integration in v1, New Identity under a proxy only rotates the browser session, never requests a new circuit). **Confirmed by e2e test** (`tests/e2e/proxy.spec.ts`, SOCKS5 and HTTP): `proxyBypassRules: ''` still leaves `localhost`/loopback destinations routed **directly, not through the proxy** — this is Chromium's own default bypass behavior, not a bug in this app's config, but it means a page that somehow causes a request to `localhost` (e.g. a misconfigured local service) is not covered by the proxy's guarantee. |
| Plaintext DNS to a network-level observer, when the proxy is off | **Mitigated when DNS-over-HTTPS mode is enabled**         | Off by default, never persisted (see ADR 0010). When enabled, `app.configureHostResolver()` restricts lookups to DoH against a chosen resolver (Quad9 or Mullvad — no Google/Cloudflare default). Greyed out in the UI while any proxy is on (DNS for tab traffic already resolves at the proxy in that case, for socks5/http/https alike — ADR 0007 decision 3, ADR 0012), but the selection itself is preserved, not reset. **Honestly scoped, not overclaimed:** CI does not assert at the packet level that queries actually leave as HTTPS to the chosen provider rather than plaintext port 53 — this sandboxed environment has no root/netns access for packet capture. What CI does assert: the toggle reaches the main process and is reflected in status (`tests/e2e/dns.spec.ts`), and the resolver-config mapping is unit-tested. A maintainer wanting packet-level proof should manually run `tcpdump -i any port 53` while toggling the setting.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Websites fingerprinting/tracking during a session                | Out of scope for v1                                       | Anti-fingerprinting (canvas, WebGL, font enumeration spoofing) is a distinct, larger problem from local footprint elimination and is explicitly out of scope for v1.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Third-party ad/tracker network requests                          | **Mitigated by default**                                  | Filter-list blocking via the @ghostery/adblocker engine over bundled EasyList + uBlock Origin snapshots (ADR 0013) — **on by default**, the one deliberate exception to this project's usual off-by-default rule for privacy toggles. `onBeforeRequest` cancels matching requests with full ABP/uBO semantics (`$third-party`, `$domain=`, `@@` exceptions, `$important`, `$redirect`, `$csp`) and PSL-correct party classification (tldts). Cosmetic `##` rules hide in-page ad slots and `##+js(...)` scriptlet rules execute in the page (the data-driven mechanism that defeats same-origin YouTube ads). CNAME uncloaking remains deferred. Lists ship bundled and are refreshed only at release time — no runtime filter-list downloads.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| OS swap/hibernation files leaking page content                   | **Yes**                                                   | This is the most common way "zero footprint" apps quietly fail. See Section 3 and known limitations below — it is a documented gap, not something the app can fully close from inside a userspace process.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| Crash reports / telemetry phoning home                           | **Yes**                                                   | Chromium and most app frameworks do this by default, writing to both disk and network.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |

**v1 priority:** optimize for "nothing recoverable from disk once the
process exits." Anti-fingerprinting and network anonymity are v2/v3
candidates, not v1 blockers.

## 2a. The self-audit panel is transparency, not a new mitigation

The start page's self-audit panel (main process, results delivered over the
existing IPC bridge as plain data — the shell renderer gains no new
capability) surfaces a live view of several rows in the table above: is
`userData` really on tmpfs right now, is the tab session partition really
non-persistent, is the HTTP cache switch really active, and so on. It does
not protect anything by itself — every check it performs is read-only and
every guarantee it reports already exists independently of the panel. Its
purpose is turning "trust the CI run and the source" into "watch it happen
in your own instance." Each row is explicit about its own evidentiary
weight: **checked now** means verified in that running process at that
instant; **enforced by CI** means the guarantee has no reliable Electron 43
runtime signal (the crash-reporter row, for one — no API on
`crashReporter` distinguishes "started" from "never started";
`research/cleanup-and-exit.md` §21) and depends on the CI lint rule instead.
Presenting a CI-only guarantee as if it were freshly checked would be
exactly the overselling this document exists to avoid.

## 3. Memory hygiene (best-effort, not perfect)

RAM is not zeroed when a process exits — the OS marks pages free but does
not scrub them until reallocated, and tooling exists to recover
recently-freed memory. Chromium's own allocator is not something this
project controls, so perfect memory scrubbing is not achievable. The
reduction implemented:

- On `window-all-closed`, explicitly `await` `clearStorageData()`,
  `clearCache()`, and `clearAuthCache()` on **both** the in-memory tab
  session and `session.defaultSession` before calling `app.exit(0)` — this
  drops Chromium's own references and lets the allocator reclaim faster.
  `defaultSession` is included even though it should only ever host the
  trusted shell UI, not visited-site content, closing a gap security review
  flagged where only the tab session was being cleared. `app.exit()` is
  used instead of `app.quit()` because `app.exit()` is immediate and skips
  `will-quit`, so any cleanup must be awaited inline before the call, not
  hooked to a lifecycle event that `app.exit()` would skip.
- No tray icon, no background mode: `window-all-closed` always quits on
  every platform (macOS's default of staying alive on window close is
  explicitly overridden).

**Honest framing:** "no recoverable trace on disk after exit" is a
supportable claim under this design. "No trace in RAM while running, or
immunity to a live memory-dump attack" is not achievable by any browser,
Amnesic Browser included, and this document does not claim otherwise.

## 4. Known limitations

Being upfront about what this project cannot fix is part of its
credibility, not a weakness to hide:

1. If the OS swaps or hibernates while the app is open, plaintext session
   data can land on disk outside the app's control. There is no
   in-process mitigation for this. Users who need this guarantee should
   disable hibernation and use encrypted swap (or no swap). Since ADR
   0004 the app checks `/proc/swaps` at startup and shows a dismissible
   warning on the start page when disk-backed swap is active (zram is
   exempt — compressed RAM never reaches disk). The warning is
   informational; it does not and cannot close the gap.
2. Live RAM forensics while the app is running can recover data — true of
   every browser, not a defect specific to this one.
3. Network-level observers (ISP, router owner) can see all traffic unless
   proxy mode is turned on (ADR 0007, generalized to any scheme by ADR 0012;
   off by default, never persisted). The proxy can be SOCKS5 (the Tor default)
   or an HTTP/HTTPS proxy a VPN or provider exposes; all three resolve DNS at
   the proxy and fail closed if it is unreachable. An HTTP/HTTPS proxy is a
   single trusted operator that sees your real IP — transport privacy, not
   anonymity, and never as private as Tor's relay model. A full **system** VPN
   (TUN/root/bundled tunnel) remains out of scope for v1 — "point the browser
   at a proxy you already run" is the in-charter slice (ADR 0012). Even with a
   proxy on, Chromium's default proxy bypass still routes `localhost`/loopback
   destinations directly rather than through the proxy — confirmed empirically
   in `tests/e2e/proxy.spec.ts`, not just assumed from docs. With the proxy
   off, DNS-over-HTTPS mode (ADR 0010, also off by default) can restrict
   lookups to an encrypted resolver, but this is not verified at the packet
   level in CI (no root/netns access in this sandboxed environment) — see the
   DNS row above for what is and isn't asserted.
4. Websites can still fingerprint/track a visitor during the session —
   local footprint elimination is not the same problem as
   anti-fingerprinting, and this project does not attempt the latter in
   v1.
5. `userData`-to-RAM redirection is not uniform across platforms: trivial
   on Linux (`/dev/shm` is tmpfs by default), materially weaker on macOS
   (no default writable tmpfs; requires shelling out to create a RAM disk
   or falling back to per-subsystem disabling plus a securely-deleted
   temp dir) and on Windows (no native tmpfs; realistic v1 approach is
   per-subsystem disabling as the primary defense, with anything that
   must be written going to `os.tmpdir()` and being securely overwritten,
   not just unlinked, before deletion — a plain delete leaves the
   underlying data blocks recoverable on disks without TRIM/encryption).
6. The crash-reporter guarantee (Section 2) depends on an enforced CI
   lint rule, not a runtime flag. If that rule is ever removed, weakened,
   or bypassed in a future PR, the guarantee silently disappears with no
   other signal.
7. The WebRTC mitigation (Section 2) is layered and each layer has a gap
   the others cover; it is not a single verifiable kill-switch, because
   Chromium does not expose one.
8. Safe Browsing's code path is present in the binary even though it is
   functionally inert by default (see Section 2) — a future change that
   adds a `GOOGLE_API_KEY` for any reason (even unrelated to Safe
   Browsing) would silently reactivate it. This is a standing constraint
   on any future dependency or feature work, not just a one-time check.
9. Content blocking (ADR 0013) bundles static filter-list snapshots, so new
   trackers may slip through until the next release refresh. The engine's two
   `@ghostery/adblocker/*` IPC channels are invokable from any page frame in a
   blocking session: a hostile page can, at most, ask for another hostname's
   cosmetic rules to be applied to itself (self-inflicted hiding) or probe
   whether the mutation observer is enabled — it cannot extract data, reach
   other frames, or make the main process throw (handlers validate input and
   fail closed to "inject nothing"). Scriptlets from the bundled
   `ubo-resources.json` execute in the page's main world by design. Request
   classification errors fail open (never stall a load). Extended/procedural
   cosmetic selectors and CNAME uncloaking are not enabled — short of full
   uBlock parity.
